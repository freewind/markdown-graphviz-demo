  -> "商量表现形式"

由于使用文本来表达图形，所以让我们可以把它放到版本控制工具中，并可以更好的Review和修改。当然，这需要我们学习和掌握DOT及相关的语言。

看一下这里的例子<http://www.gravizo.com/#samples>，原来我们可以用各种各样的代码语言来描述出这么复杂的图形，一旦掌握，可比画出来的要容易得多。
